# Computer Care Skills（コンピューターケアスキル）

[中文](README.md) | [English](README.en.md) | **日本語**

ローカルコンピューターの管理・メンテナンス・自動化のためのスキル群です。診断、修復、クリーンアップ、パス処理、ワークステーション調整、プロセス監視、ファイル監視などに対応し、macOS / Windows / Linux で動作します。

基本原則：まず診断し、それから提案する。まず読み取り専用のチェックを行い、その後で元に戻せる最小限の変更を実行する。ファイル削除、システム設定の変更、プロセス停止、ネットワーク設定、常駐型の自動化を行う場合は、最初に範囲・リスク・ロールバック方法・検証手順を明示します。

## スキル構成

```
Computer Care
├─ 診断と修復      ─── computer-health    network-optimize    permissions-fix
├─ クリーンアップ  ─── storage-clean      path-convert        desktop-tune
├─ 監視と自動化    ─── processes-monitor  folder-watch
└─ Codex 自己管理  ─── codex-optimize
```

## インストール

```bash
npx skills add VintLin/computer-care-skills
```

インストール可能なスキルの一覧：

```bash
npx skills add VintLin/computer-care-skills --list
```

単一スキルのみインストール：

```bash
npx skills add VintLin/computer-care-skills --skill computer-health
```

## スキル一覧

### 診断と修復

| スキル | 目的 | 主なユースケース |
| --- | --- | --- |
| `computer-health` | パソコンのヘルスチェックとパフォーマンス診断 | 動作が遅い、メモリ負荷が高い、スワップ/ページファイルの肥大化、CPU飽和、サーマルスロットリング、ディスクI/O、コードのベンチマーク/プロファイリング（macOS / Windows / Linux） |
| `network-optimize` | ネットワーク診断と最適化 | 通信速度低下、高レイテンシ、パケットロス、DNS遅延、Wi-Fi/イーサネット、MTU、VPN/プロキシ/TUNの競合（macOS / Windows / Linux） |
| `permissions-fix` | macOSの権限修復 | Operation not permittedエラー、フルディスクアクセス、TCC権限の不整合、バックグラウンドタスクの権限拒否 |

### クリーンアップとメンテナンス

| スキル | 目的 | 主なユースケース |
| --- | --- | --- |
| `storage-clean` | ストレージのクリーンアップ | ディスク容量不足、キャッシュの肥大化、ダウンロードフォルダの整理、Docker/Podman/WSLの占有、パッケージマネージャーのキャッシュ（macOS / Windows / Linux） |
| `path-convert` | クロスプラットフォームのパス変換 | Windows UNC ↔ macOS smb://、/Volumes パス、パスを開く、ファイル一覧 |
| `desktop-tune` | macOSワークステーションの調整 | 常時給電、バッテリー寿命、スリープ/復帰の異常、USB/オーディオデバイスの切断、ディスプレイのスリープ |

### 監視と自動化

| スキル | 目的 | 主なユースケース |
| --- | --- | --- |
| `processes-monitor` | 長時間プロセスの監視 | バックグラウンドのコマンド・スクリプト・ビルド・ダウンロードのハートビート、ログ、タイムアウト、状態判定 |
| `folder-watch` | フォルダ変更の自動化 | ディレクトリの監視、変更時のコマンド実行、ローカルサービスの再起動、バックアップや通知のトリガー |

### Codex 自己管理

| スキル | 目的 | 主なユースケース |
| --- | --- | --- |
| `codex-optimize` | Codexのローカル状態メンテナンス | `.codex` セッション・ログ・ワークツリー・設定の肥大化解消、スレッドメタデータの肥大化解消、プロバイダーメタデータの同期 |

## 使い方

インストール後、直面している問題をそのまま説明するだけです：

```text
Macのディスク容量がもう少しでいっぱいです。安全に削除できるものを探してください。
```

```text
\\server\share\project を macOS Finder で開けるパスに変換してください。
```

```text
VPN接続中はブラウザが遅いのに、ターミナルのcurlは普通です。診断してください。
```

```text
Macのメモリがいっぱいに見えて、コードの実行も遅いです。ヘルスチェックしてボトルネックを見つけてください。
```

```text
ダウンロードフォルダを監視して、新しいファイルが来たらスクリプトを実行してください。
```

## 動作の仕組み

すべてのスキルは同じ処理フローに従います：

1. 症状、環境、パス、エラーメッセージ、メトリクスを収集する。
2. まず読み取り専用の診断を実行し、システム変更やファイル削除はまだ行わない。
3. 候補となる操作をリスクでランク付けし、推奨する理由・非推奨の理由を説明する。
4. 影響の大きい操作については、事前に目的と範囲を確認する。
5. 実行後、同じ指標やコマンドで結果を検証する。
6. 結論、原因、リスク、次のステップ、ロールバック手順を出力する。

## 安全の境界線

明示的に目的と範囲が確認されない限り、これらのスキルはデフォルトで以下を実行しません：

- ユーザーファイルの削除、ゴミ箱を空にする、不可逆な一括削除。
- プロセスの停止、サービスの再起動、ソフトウェアのアンインストール、常駐タスクの改変。
- DNS、MTU、ルート、VPN、プロキシ、システムネットワークサービスの変更。
- macOS TCC権限のリセット、権限データベースの編集、システム保護のバイパス。
- 影響の大きいクリーンアップ（Docker全体のprune、モデルキャッシュの削除、ツールチェーンの削除）。
- セキュリティ監視、フォレンジック監視、不正アクセスを目的とした自動化の構築。
